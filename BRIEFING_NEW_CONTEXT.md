# BRIEFING: Alfa Tracker — Переход на бэкенд + БД + Авторизация

## Контекст проекта

**Проект:** Alfa Tracker — инструмент для сотрудников Альфа-Банка.  
**Текущее состояние:** Работающий фронтенд — один файл `index.html` (~1700 строк), чистый JS без фреймворков, без сборки. Хранилище сейчас — `localStorage` браузера.  
**Рабочая директория:** `e:\AI\Alfa-project\`

---

## Что нужно построить

Добавить полноценный бэкенд, чтобы:
1. У каждого из 40 сотрудников была **своя учётная запись** (логин/пароль).
2. Встречи, которые сотрудник вводит через `index.html`, **сохранялись в SQLite-базу** (не в localStorage).
3. История встреч и статистика **не слетали** при обновлении страницы / смене устройства.
4. Приложение будет **доступно всем 40 сотрудникам** через хостинг (не только локально).

## Schema БД (файл `database\schema.sql` уже создан)

```sql
CREATE TABLE sotrudniki (
    id_sotrudnika TEXT PRIMARY KEY,  -- СОТР-001..СОТР-040
    fio           TEXT NOT NULL,
    otdel         TEXT NOT NULL,
    dolzhnost     TEXT NOT NULL,
    email         TEXT NOT NULL UNIQUE
);

CREATE TABLE vstrechi (
    id_vstrechi      TEXT    PRIMARY KEY,
    id_sotrudnika    TEXT    NOT NULL REFERENCES sotrudniki(id_sotrudnika),
    klient           TEXT    NOT NULL,
    data_vstrechi    TEXT    NOT NULL,   -- YYYY-MM-DD
    tip_vstrechi     TEXT    NOT NULL,
    kratkoe_opisanie TEXT    NOT NULL,
    rezultat         TEXT    NOT NULL,
    ocenka           INTEGER NOT NULL CHECK(ocenka BETWEEN 1 AND 10)
);
-- + индексы и VIEW для статистики (см. schema.sql)
```

> **Встречи в БД НЕ предзаполнены.** Только структура. Встречи будут поступать из приложения.

## Задача для нового контекста — ВЫПОЛНИТЬ ВСЁ ПО ПОРЯДКУ

### Шаг 1 — `package.json` + установка зависимостей
```json
{
  "name": "alfa-tracker",
  "version": "1.0.0",
  "main": "server.js",
  "dependencies": {
    "express": "^4.18.2",
    "better-sqlite3": "^9.4.3",
    "bcryptjs": "^2.4.3",
    "jsonwebtoken": "^9.0.2",
    "cookie-parser": "^1.4.6"
  }
}
```

### Шаг 2 — `server.js` (Express + SQLite)
Реализовать следующие API-эндпоинты:

| Метод | Путь | Описание |
|---|---|---|
| POST | `/api/login` | Авторизация (fio + password) → JWT-токен в httpOnly cookie |
| POST | `/api/logout` | Выход, удаление cookie |
| GET | `/api/me` | Текущий пользователь (из токена) |
| GET | `/api/vstrechi` | Список встреч текущего сотрудника |
| POST | `/api/vstrechi` | Сохранить новую встречу |
| DELETE | `/api/vstrechi/:id` | Удалить встречу |
| GET | `/api/stat` | Статистика по текущему сотруднику |
| GET | `/api/admin/vstrechi` | ВСЕ встречи (только для роли admin) |

### Шаг 3 — `database\init.js`
Скрипт инициализации БД:
- Применяет `schema.sql`
- Вставляет 40 сотрудников (INSERT OR IGNORE)
- Создаёт пользователей с дефолтным паролем `alfa2024` (хэш bcrypt) — при первом входе сотрудник меняет пароль

### Шаг 4 — Адаптация `index.html`
- Добавить экран логина (поверх текущего приложения)
- Заменить обращения к `localStorage` (`alfa_h`) на `fetch('/api/vstrechi')`
- При `completeMtg()` — POST запрос на сервер вместо записи в localStorage
- При загрузке `renderHome` — GET запрос истории с сервера
- Токен хранится в httpOnly cookie (браузер отправляет автоматически)

### Шаг 5 — Проверка и запуск
```bash
node database/init.js   # инициализация БД
node server.js          # запуск сервера на порту 3000
```

## Поля объекта `meeting` из `index.html` (важно для маппинга в БД)

Основные поля, которые нужно сохранять:
- `clientName` — имя клиента
- `primaryType` — тип главного продукта (ДК, КК1, КК2, КН, Х5, DC ИНВ, Селфи ДК, Селфи КК, ДК+Детская)
- `AW` — тип встречи (OP = очная)
- `AV` — итоговое вознаграждение (рассчитывается при завершении)
- `AA` — количество БС (bancassurance)
- `completedAt` — дата завершения (ISO строка)
- `status` — `COMPLETED`
