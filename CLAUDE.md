# Alfa Tracker

Однофайловое SPA (`index.html`, ~915 строк) для трекинга встреч сотрудника Альфа-Банка и расчёта вознаграждения. Без сборки, без зависимостей. Хранилище — `localStorage` (ключ `alfa_h`).

## Структура `index.html`
- `<style>` — UI (красная тема Альфы, мобильный layout, max-width 480px).
- `<header>` — заголовок + кнопка `← Назад` (вызывает `goBack()`).
- `<script>`:
  - `newMeeting()` — модель встречи. Поля по буквам Excel-колонок (F,G,H... AV,AW). Комментарии в коде поясняют группы: ДК, КК, Селфи, Кросс, Доп.продажи, Приложение, Флаги.
  - `bsRate(totalBS)` — ставка БС от месячного объёма (270 / 430 / 570).
  - `monthlyBSTotal()` — сумма `AA` из `history` за текущий месяц.
  - `calcEarnings(m, forceBSRate?)` — итоговая формула, умножается на `0.87` (НДФЛ).
  - State: `meeting`, `screen` (`home`/`meeting`/`result`), `history`, `navStack`.
  - Навигация: `navigate(s, resetMeeting?)`, `goBack()`, `goHome()`, `startMtg()`, `completeMtg()`, `setStage(s)`.
  - Render: один `render()` диспатчит на `renderHome` / `renderMeeting` / `renderResult` по `screen`. Этап встречи — `meeting.AW` (`OP`, ..., `COMPLETED`).
  - DOM-хелперы: `el`, `btn`, `card`, `grid`.

## Доменные правила (важно не сломать)
- `meeting.E = F+G+H+I+J+K+L` пересчитывается в `completeMtg()`.
- `AV` — итоговое вознаграждение, сохраняется в history только при `COMPLETED`.
- Ставка БС — динамическая по месячному накоплению, формула чувствительна к порядку расчёта.
- История — массив объектов meeting; не менять схему полей без миграции `localStorage`.

## Соглашения по правкам
- Минимальные точечные правки, без рефакторинга/абстракций.
- Без новых файлов и зависимостей.
- UI-тексты на русском.
- Комментарии — только если "почему" неочевидно.
