Бот предназначен для сбора детальной отчетности агентов о проведенных встречах с клиентами. Основная задача: пошаговый сбор данных о проданных продуктах (основных и дополнительных), их статусах активации и итоговый расчет результативности.

## 3. Логика ветвления (User Flow)

### Шаг 1: Инициация
* Команда: `Начать встречу`

### Шаг 2: Основной продукт
* **Действие:** Выбор из списка (ДК, КК1, КК2, Х5, ЗПК, RE, СИМ, Семейный, ДК+Детская, Селфи n2b, Селфи ДК, Селфи КК, КН, Установка МП, Отказ банка).
* **Исключения:**
    * `Отказ банка`: Прямой переход на **Шаг 9 (Финал)**.
    * `Установка МП`: Завершение встречи. Сохранение только этого действия.
    * `Х5`: Переход к Шагу 3, затем пропуск Шагов 4-7 (сразу к Шагу 8).

### Шаг 3: Статус основного продукта
* **Кнопки:** `ВА+ ТР+`, `ВА+ ТР-`, `В+`

### Шаг 4: Кросс-продажа (Доп. продукт)
* **Действие:** Выбор из списка (КЛ/ККК, КДК к ДК, Селфи n2b, Селфи ДК, Селфи КК, Кросс СИМ, Кросс Детская, Пропустить).
* **Логика:** Если выбран продукт -> переход к **Шагу 5**. Если "Пропустить" -> переход к **Шагу 6**.

### Шаг 5: Статус кросс-продукта
* **Кнопки:** `ВА+ ТР+`, `ВА+ ТР-`, `В+`

### Шаг 6: Инвестиции и Сервисы
* **Вопрос 1:** БС или Инвесткопилка? (Кнопки: БС, Инвесткопилка, Обе, Никакая).
* **Вопрос 2:** ЦП или Вок? (Кнопки: ЦП, Вок, ЦП и Вок, Никакая).

### Шаг 7: Дополнительные продукты (Мультивыбор/Цикл)
* **Кнопки:** Смарт, Комбо 1Р, Кэш, Защитник, ПДС, Пенсия, Установка МП Х5, **Далее**.
* **Логика:** Пользователь может выбрать любое количество продуктов. После каждого нажатия список продуктов обновляется (продукт добавляется в отчет). Переход дальше только по кнопке **"Далее"**.

### Шаг 8: Мобильное приложение
* **Кнопки:** Android установил, Android не установил, iPhone установил, iPhone не установил.

### Шаг 9: Финальное действие
* **Варианты:**
    1.  `Завершить встречу`: Сохранение всех данных в БД, генерация итогового текста отчета.
    2.  `Ещё продукт`: Сохранение данных по текущему продукту и возврат на **Шаг 2** (для добавления второго основного продукта тому же клиенту).
    3.  `Удалить встречу`: Полная очистка данных текущей сессии.

## 4. Технические рекомендации для разработки
1.  **State Management:** Использовать FSM (Finite State Machine) для отслеживания текущего шага агента.
2.  **Data Storage:** Каждая встреча должна иметь уникальный `meeting_id`. При нажатии "Ещё продукт" данные добавляются в ту же встречу.
3.  **Calculation:** Реализовать функцию, которая при сохранении сопоставляет выбранные продукты с `Tariff_DB` и выдает сумму "Итого".
